# CryptoSafe Manager (Sprint 1–3)

CryptoSafe Manager — настольный менеджер секретов и паролей с локальным зашифрованным хранилищем, модульной архитектурой и расширяемым графическим интерфейсом.

Проект развивается по спринтам.  
На текущем этапе реализованы:

- **Sprint 1** — архитектурный фундамент
- **Sprint 2** — аутентификация и управление ключами
- **Sprint 3** — полноценный CRUD хранилища и шифрование записей через **AES-256-GCM**

> ⚠️ Важно:
> - В **Sprint 1** использовалась криптографическая заглушка XOR.
> - В **Sprint 2** были добавлены Argon2id, PBKDF2, кэширование ключей и смена мастер-пароля.
> - Начиная с **Sprint 3**, новые записи шифруются индивидуально через **AES-256-GCM**.
> - Для старых записей из предыдущих спринтов поддерживается совместимость и миграция к новому формату.

---

## Видение проекта

Цель проекта — безопасное локальное хранилище учётных данных и секретов с возможностью дальнейшего расширения:

- надёжное хранение логинов, паролей, заметок и ссылок
- полноценная криптография: AES-GCM + KDF + управление ключами
- журнал аудита и система событий
- безопасный буфер обмена
- авто-блокировка по неактивности
- поиск, фильтрация, теги, категории
- backup/restore, импорт/экспорт
- подготовка к упаковке и релизу

## Что реализовано по спринтам

---

## Sprint 1

### Основные результаты
- Создана модульная архитектура проекта
- Добавлена схема SQLite
- Реализованы:
  - `ConfigManager`
  - `StateManager`
  - `EventBus`
  - `AuditLogger`
- Подготовлен GUI shell
- Добавлены виджеты:
  - `PasswordEntry`
  - `SecureTable`
  - `SettingsDialog`
- Подготовлены тесты и CI

### Ограничения Sprint 1
- вместо реального шифрования использовался XOR placeholder
- не было полноценной аутентификации
- не было управления мастер-паролем

---

## Sprint 2

### Основные результаты
- Реализована аутентификация по мастер-паролю
- Добавлено хэширование через **Argon2id**
- Добавлено формирование encryption key через **PBKDF2-HMAC-SHA256**
- Добавлен **SecureKeyCache**
- Добавлен login dialog
- Добавлена смена мастер-пароля
- Реализована ротация ключей
- Обновлена таблица `key_store`
- Добавлена password policy:
  - минимум 12 символов
  - uppercase / lowercase / digit / special
  - защита от слабых шаблонов

### Ограничения Sprint 2
- записи всё ещё шифровались старой XOR-заглушкой
- подготовка к реальному шифрованию была завершена, но сам AES-GCM ещё не использовался

---

## Sprint 3

### Основные результаты
- Реализовано **индивидуальное шифрование записей через AES-256-GCM**
- Каждая запись хранится как:
  - `nonce (12 байт)`
  - `ciphertext + tag`
- Все поля записи сериализуются в JSON и шифруются целиком
- Добавлен `EntryManager` для CRUD-операций:
  - `create_entry`
  - `get_entry`
  - `get_all_entries`
  - `update_entry`
  - `delete_entry`
- Добавлен безопасный генератор паролей
- Добавлен поиск по нескольким полям
- Добавлена реальная таблица записей
- Добавлен диалог создания и редактирования записи
- Добавлено мягкое удаление в `deleted_entries`
- Реализована совместимость со старыми записями Sprint 1–2

### Что делает Sprint 3 особенно важным
Sprint 3 переводит проект из состояния “архитектурного фундамента” в состояние **реально работающего локального менеджера паролей**, где:
- есть аутентификация,
- есть ключевая инфраструктура,
- есть шифрование записей,
- есть CRUD,
- есть пользовательский интерфейс для работы с данными.

## Архитектура проекта

Проект организован по MVC-подобной схеме.

### Слои
- **View** → `src/gui/`
- **Core / Controller** → `src/core/`
- **Database / Model** → `src/database/`

### Поток работы
1. GUI инициирует действие пользователя  
2. Core обрабатывает бизнес-логику  
3. AuthenticationManager управляет доступом к encryption key  
4. EntryManager выполняет CRUD  
5. EncryptionService шифрует/расшифровывает payload записи  
6. Database сохраняет данные  
7. EventBus публикует события  
8. AuditLogger пишет аудит
